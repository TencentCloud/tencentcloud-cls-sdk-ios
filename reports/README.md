# 📊 分析报告目录

## 目录说明

此目录用于存放项目的各类自动生成的分析报告文件，包括但不限于：

- 📝 测试报告分析 (TEST_REPORT_ANALYSIS.md)
- 🌐 HTML格式分析报告 (test-report-analysis.html)
- 🔍 代码审查报告 (code-review-report-*.html)
- 📈 性能分析报告
- 🐛 问题追踪报告
- 其他自动生成的分析文档

## 🚫 Git 策略

**此目录已被添加到 `.gitignore` 中，目录下的所有文件（除 `.gitkeep`）都不会被提交到 Git 仓库。**

理由：
- 这些报告文件通常体积较大
- 内容是自动生成的，可以随时重新生成
- 避免版本控制系统中积累大量历史版本

## 📂 文件组织建议

建议按照以下方式组织报告文件：

```
reports/
├── README.md                           # 本说明文件
├── .gitkeep                            # 保持目录结构
├── code-review-report-YYYYMMDD.html    # 代码审查报告（按日期命名）
├── test-report-analysis.html          # 测试报告分析（HTML格式）
├── TEST_REPORT_ANALYSIS.md            # 测试报告分析（Markdown格式）
├── performance-report-YYYYMMDD.html   # 性能分析报告
└── ...                                # 其他分析报告
```

## 💡 使用建议

1. **本地查看**: 这些报告仅用于本地查看和分析
2. **定期清理**: 建议定期清理过期的报告文件
3. **重要报告**: 如需长期保存，可以：
   - 上传到专门的文档管理系统
   - 存档到团队共享目录
   - 导出为 PDF 后保存

## 📌 注意事项

- ✅ 此目录可以随时删除和重建
- ✅ 报告文件命名建议包含日期，便于追踪
- ❌ 请勿在此目录存放需要版本控制的重要文档
- ❌ 请勿存放包含敏感信息的报告
